# 🩺 NCDs-Screening (ระบบคัดกรองเบาหวาน-ความดันโลหิตสูง)

ระบบเว็บแอปพลิเคชันคัดกรองโรคไม่ติดต่อเรื้อรัง (NCDs) เบาหวานและความดันโลหิตสูง พร้อมระบบประเมินความพฤติกรรมการบริโภคอาหาร (หวาน มัน เค็ม) และการวิเคราะห์กลุ่มเสี่ยงด้วยปิงปอง 7 สี

---

## 🚀 วิธีแก้ไขปัญหาหน้าเว็บขาวบน GitHub Pages (ยังไม่แสดงผล)

สาเหตุที่หน้าเว็บขึ้นเป็นหน้าขาวและแจ้งเตือน Error `main.tsx 404` ใน Console เป็นเพราะ **GitHub Pages กำลังดึงไฟล์ Source Code ดิบ (index.html หลัก) ไปแสดงผลโดยตรง** ซึ่งบราวเซอร์ไม่สามารถรันไฟล์ TypeScript (`.tsx`) ได้

ทางเราได้สร้างไฟล์ระบบสำหรับสั่งรันและ Build อัตโนมัติด้วย **GitHub Actions** ไว้ให้ท่านเรียบร้อยแล้ว ในห้อง `.github/workflows/deploy.yml`

เพื่อเปิดใช้งานระบบการทำงานอัตโนมัตินี้ ให้ท่านเข้าไปตั้งค่าในเว็บไซต์ GitHub ดังนี้ครับ:

1. **เปิดหน้า Repository** ของท่านใน GitHub (ลิงก์: https://github.com/langumoblie-wq/NCDs-Screening)
2. ไปที่เมนู **Settings** (รูปฟันเฟืองด้านบนขวา)
3. เมนูด้านซ้าย เลือก **Pages** (ภายใต้หัวข้อ Code and automation)
4. ในหัวข้อ **Build and deployment** -> ตรงหัวข้อ **Source** ให้กดเปลี่ยนจากเดิมที่เป็น `Deploy from a branch` ให้เป็น **`GitHub Actions`**
5. **ทำการอัปเดตโค้ดหรือคอมมิตใหม่**: เมื่อเปลี่ยนเป็น GitHub Actions แล้ว ระบบ GitHub จะเริ่มทำงานและคอมไพล์โค้ดจากระบบ Vite ลงไปในเซิร์ฟเวอร์ และเปิดใช้งานหน้าเว็บให้อย่างสมบูรณ์แบบโดยอัตโนมัติภายใน 1-2 นาทีครับ!

## 🛠️ รายละเอียดการทำงานของระบบและฟังก์ชันที่อัปเดตล่าสุด

เราได้ทำการตรวจสอบและปรับปรุงระบบให้มีความสมบูรณ์แบบและป้องกันอาการจอขาว (Blank Screen) เรียบร้อยแล้ว:

1. **ระบบป้องกันข้อมูลเป็นโมฆะ (Null/Undefined Safety)**
   - เพิ่มระบบตรวจสอบข้อมูลย้อนหลัง ป้องกันปัญหากรณีฐานข้อมูลเกิดปัญหาการเชื่อมต่อหรือข้อมูลในบางแถวเป็นค่าว่าง (Null) เพื่อป้องกันหน้าจอดับหรือขาวขณะกดบันทึกข้อมูล
   - เพิ่มระบบออฟไลน์สมบูรณ์แบบ (`localStorage Backup`) หากขาดการเชื่อมต่ออินเทอร์เน็ตหรือ Supabase ขัดข้อง ระบบจะสลับไปบันทึกข้อมูลและดึงข้อมูลมาแสดงผลในเครื่องผู้ใช้ทันที เพื่อไม่ให้งานบริการสะดุด

2. **ระบบ Deployment อัตโนมัติ**
   - ตั้งค่า `base: '/NCDs-Screening/'` ใน `vite.config.ts` ให้ตรงกับชื่อโปรเจกต์ของท่านบน GitHub Pages เรียบร้อยแล้ว
   - เตรียมไฟล์ Workflow สำหรับบิวด์และอัปโหลดขึ้น GitHub Pages โดยอัตโนมัติเมื่อมีการ push ข้อมูล
